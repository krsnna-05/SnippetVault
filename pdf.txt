01

Project Overview

You will build SnippetVault — a public code snippet manager. It is a tool developers use to save, tag, search, and
share reusable bits of code. Think of it as a lightweight, focused alternative to GitHub Gists, with a clean public
profile and rich sharing features.
The app has two distinct modes: an authenticated workspace where a user manages their own snippets, and a
public-facing surface where anyone can browse, view, and copy shared snippets without logging in.

What you are building
•​ Email/password authentication with protected and public routes
•​ A personal dashboard to create, edit, search, and filter your snippets
•​ A public profile page showing all snippets a user has made public
•​ Four sharing mechanisms: public URL, share with specific users, image export, and clipboard link
•​ Syntax-highlighted code preview with copy-to-clipboard feedback
Scope note

Use Supabase Auth for authentication — do not build your own auth system. Row Level Security (RLS)
policies on your tables are required to protect private snippets.

requesting user. INSERT/UPDATE/DELETE allowed only if user_id = auth.uid(). snippet_shares:
SELECT/INSERT/DELETE allowed only for the snippet owner (user_id = auth.uid()).

State Management

Server state lives in TanStack Query. UI state lives in Zustand. These must not be mixed.

TanStack Query — Server State
•​
•​
•​
•​
•​
•​

useQuery(['snippets', userId]) — fetch the current user's snippets with tags. staleTime: 60s.
useQuery(['snippet', id]) — single snippet for the detail panel
useQuery(['shares', snippetId]) — list of users a snippet is shared with
useMutation (create / update) — on success, invalidate ['snippets', userId]
useMutation (delete) — optimistic removal from list cache; roll back on error with a toast
useMutation (addShare / removeShare) — invalidate ['shares', snippetId] on settle

Edge Cases to Handle

Auth Edge Cases
•​ Expired session: user should be silently redirected to /login, not shown a broken page
•​ Signing up with an already-registered email: show 'An account with this email already exists'
•​ Accessing /dashboard while logged out: redirect to /login with a ?redirect= param so they return after login
Sharing Edge Cases
•​ Sharing with your own email: show 'You cannot share a snippet with yourself'
•​ Sharing with an email not in the system: show 'No account found with that email'

•​ Trying to export image on a very long snippet: cap the export at 50 lines and add a '...truncated' indicator
•​ Making a snippet private after it has been shared via URL: the public URL must immediately return 404 —
but shared-with users retain access

General UI Edge Cases
•​ Copy button: works on both http and https; has a textarea fallback for older browsers
•​ Filter with no results: friendly empty state + 'Clear filters' button
•​ Failed delete: optimistic rollback + toast with retry option
•​ Failed image export: toast with the error message, button returns to default state

Bonus A — View Count
Track how many times a public snippet page (/s/[id]) has been viewed. Show the count on the detail panel and the
public page. Increment must happen server-side (a Server Action or API route) — not from the client directly to
avoid RLS bypass.

Bonus B — Keyboard Navigation
Arrow keys navigate between snippet cards. Enter opens the detail panel. Escape closes it. Tab moves focus into
